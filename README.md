# VMS

**Vehicle Management System (VMS)** built using a **microservices architecture** with **NestJS** for the backend, **Angular** for the frontend, and **Redis** for caching. The system supports both **REST and GraphQL APIs**, as well as **batch jobs** for automated tasks.

---

## 🏗 Architecture

- **Frontend:** Angular  
- **Backend:** NestJS (Microservices)  
- **API Types:** REST + GraphQL  
- **Database & Caching:** Redis (for caching temporary data or jobs)  
- **Background Jobs:** Import and Export Excels  

The system follows a **modular microservices approach**, separating concerns for scalability and maintainability.

---

## ⚡ Features

- Full CRUD operations for vehicles, services, and other resources  
- REST and GraphQL endpoints for flexible API access  
- Background batch jobs for processing tasks asynchronously  
- Caching layer using Redis for improved performance  
- Microservices design enabling independent scaling of modules
